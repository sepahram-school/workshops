# Iceberg Benchmark Plan — `benchmark_iceberg.py`

## Goal

Create a `benchmark_iceberg.py` script that benchmarks Apache Iceberg table operations on S3-compatible storage (MinIO, RustFS, libreFS, SeaweedFS), using **file-based metadata management** (Hadoop catalog).

**Engine**: PyIceberg (Hadoop catalog) + Daft (data I/O) + DuckDB (analytical reads via `iceberg_scan()`)
**Metadata**: File-based — `version-hint.txt` + `vN.metadata.json` stored on S3 alongside data
**Results dir**: `results_iceberg/{mode}/{size}/{target}/run-N.json`

## 2. Create `benchmark_iceberg.py`

### Architecture

```
PyIceberg (Hadoop catalog)  →  writes Parquet + metadata files to S3
Daft                        →  DataFrame generation, data I/O for PyIceberg append()
DuckDB (iceberg_scan)       →  reads from S3 via metadata files
```

### Hadoop Catalog Setup

```python
catalog = load_catalog(
    "benchmark_hadoop",
    type="hadoop",
    warehouse=f"s3://iceberg-warehouse-{target}/",
    s3.endpoint=..., s3.access-key-id=..., s3.secret-access-key=...
)
```

### Schema

**events** (main, partitioned by `event_time` day):
- `id` (Integer), `category` (String), `value` (Float), `event_time` (Timestamp)

**orders** (secondary):
- `order_id` (Integer), `customer_id` (Integer), `product` (String), `amount` (Float), `order_date` (Timestamp)

### 9 Benchmark Modes

| Mode | What It Measures |
|------|-----------------|
| `write-only` | Bulk append throughput + small streaming TPS |
| `read-only` | Scan, aggregate, point query via DuckDB iceberg_scan() |
| `mixed` | Sequential write → read |
| `heavy` | Concurrent writes + reads for 30s |
| `time-travel` | Snapshot creation + AT VERSION query |
| `snapshots` | Metadata listing latency |
| `change-detection` | Diff between snapshots |
| `update` | Row-level update via PyIceberg |
| `delete` | Row-level delete via PyIceberg |

### CLI

```bash
uv run python benchmark_iceberg.py --target minio --mode heavy
uv run python benchmark_iceberg.py --target all --sizes 1mb,32mb --runs 3
uv run python benchmark_iceberg.py --target minio --mode time-travel
```

### Per-Run Orchestration

1. Create unique S3 bucket (`benchiceberg-{target}-run{N}-{uuid}`)
2. Create Hadoop catalog → `s3://{bucket}/warehouse/`
3. Instantiate IcebergBenchmark with catalog + DuckDB
4. Execute mode-specific phase(s)
5. Save results to `results_iceberg/{mode}/{size}/{target}/run-N.json`
6. Cleanup: drop tables, close connections, delete bucket
